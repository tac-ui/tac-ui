## Critical Issues

### AUDIT-2026-0001: Form controls missing aria-invalid on error state
- **Target:** Input.tsx:34, Textarea.tsx:22, Select.tsx:48
- **Description:** When `error` is true, Input, Textarea, and Select apply visual error styling but do not set `aria-invalid="true"`. Screen readers cannot detect the error state. WCAG 1.3.1 and 4.1.2 violation.
- **Suggestion:** Add `aria-invalid={error || undefined}` to the `<input>`, `<textarea>`, and `<select>` elements.

### AUDIT-2026-0002: Radio input missing visible focus indicator
- **Target:** Radio.tsx:62
- **Description:** Hidden radio input uses `className="absolute opacity-0"` without `peer` class. The visual radio circle has no `focus-visible` ring. WCAG 2.4.7 failure.
- **Suggestion:** Change input class to `"peer sr-only"`. Add `peer-focus-visible:ring-2 peer-focus-visible:ring-[var(--primary)]/40 peer-focus-visible:ring-offset-1` to the visual circle span.

### AUDIT-2026-0003: Switch button missing visible focus indicator
- **Target:** Switch.tsx:30
- **Description:** The Switch `<button>` has no `focus-visible:` styles. WCAG 2.4.7 failure.
- **Suggestion:** Add `focus-visible:outline-2 focus-visible:outline-[var(--primary)] focus-visible:outline-offset-2`.

### AUDIT-2026-0004: Dialog, Modal, and Drawer missing aria-labelledby
- **Target:** Dialog.tsx:52, Modal.tsx:63, Drawer.tsx:76
- **Description:** All three have `role="dialog"` and `aria-modal="true"` but no `aria-labelledby` or `aria-label`. WCAG 4.1.2 violation. Title sub-components exist but are not wired up.
- **Suggestion:** Use `React.useId()` to generate a title ID, pass to Title sub-component as `id`, add `aria-labelledby={titleId}` to dialog div.

### AUDIT-2026-0005: Snackbar missing role and aria-live region
- **Target:** Snackbar.tsx:94
- **Description:** Snackbar renders notifications dynamically but has no `role` or `aria-live`. WCAG 4.1.3 failure. Toast.tsx correctly uses `role="alert"`.
- **Suggestion:** Add `role="status"` and `aria-live="polite"` (or `role="alert"` for error variants). Add `aria-atomic="true"`.

### AUDIT-2026-0006: Multiple interactive elements missing focus-visible styles
- **Target:** Tabs.tsx:81, Accordion.tsx:75, Pagination.tsx:33, Chip.tsx:35, AnimatedToggle.tsx:18, CodeBlock.tsx:28
- **Description:** Many interactive elements lack focus-visible indicators. WCAG 2.4.7 failure. Button component correctly uses `focus-visible:outline-2` as the reference pattern.
- **Suggestion:** Add `focus-visible:outline-2 focus-visible:outline-[var(--primary)] focus-visible:outline-offset-2` to all listed elements. Consider a shared `focusRing` utility.

### AUDIT-2026-0007: Tabs missing aria-controls/id linkage
- **Target:** Tabs.tsx:83
- **Description:** TabTrigger has `role="tab"` and `aria-selected` but no `aria-controls`. TabContent has `role="tabpanel"` but no `id` or `aria-labelledby`. WAI-ARIA Tabs Pattern requires these linkages.
- **Suggestion:** Set `id="tab-{value}"` and `aria-controls="tabpanel-{value}"` on TabTrigger; `id="tabpanel-{value}"` and `aria-labelledby="tab-{value}"` on TabContent.

### AUDIT-2026-0008: Accordion missing aria-controls linkage
- **Target:** Accordion.tsx:75
- **Description:** AccordionTrigger sets `aria-expanded` but no `aria-controls`. AccordionContent has no `id` or `role="region"`. WAI-ARIA Accordion Pattern requires these associations.
- **Suggestion:** Add `id="accordion-content-{value}"` to content, `aria-controls` on trigger, and `role="region"` with `aria-labelledby` on content.

### AUDIT-2026-0009: Progress missing role=progressbar and ARIA values
- **Target:** Progress.tsx:52
- **Description:** Neither variant sets `role="progressbar"`, `aria-valuenow`, `aria-valuemin`, or `aria-valuemax`. WCAG 4.1.2 A failure.
- **Suggestion:** Add `role="progressbar"`, `aria-valuenow={Math.round(percent)}`, `aria-valuemin={0}`, `aria-valuemax={100}`.

### AUDIT-2026-0010: No prefers-reduced-motion support anywhere
- **Target:** cross-cutting (all animated components)
- **Description:** Zero matches for `prefers-reduced-motion` across all components. AnimatedToggle, Accordion, Skeleton, Indicator, Button, Dialog/Modal/Drawer all animate. WCAG 2.3.3 violation.
- **Suggestion:** Add global CSS `@media (prefers-reduced-motion: reduce)` rule, or use Tailwind `motion-reduce:` utilities per element.

## High Issues

### AUDIT-2026-0011: Form control error/helper text not linked via aria-describedby
- **Target:** Input.tsx:53, Textarea.tsx:35, Select.tsx:70
- **Description:** Error and helper text spans are visually associated but not programmatically linked. Screen readers will not announce error messages.
- **Suggestion:** Generate unique ID for error/helper span, add `aria-describedby={errorId}` to form control.

### AUDIT-2026-0012: Button with iconOnly lacks enforced accessible label
- **Target:** Button.tsx:58
- **Description:** When `iconOnly` is true, no enforcement to require `aria-label`. WCAG 4.1.2 violation.
- **Suggestion:** Document requirement; consider a runtime dev warning.

### AUDIT-2026-0013: Slider missing accessible label connection and focus indicator
- **Target:** Slider.tsx:21
- **Description:** Range input not connected to label via `htmlFor`/`id`. Label is a plain `<span>`. No focus-visible styles on thumb. WCAG 1.3.1 and 2.4.7 violations.
- **Suggestion:** Use `useId()`, change span to `<label htmlFor={id}>`, add focus-visible styles to thumb.

### AUDIT-2026-0014: Combobox missing aria-activedescendant and results announcement
- **Target:** Combobox.tsx:140
- **Description:** No `aria-activedescendant` for highlighted option. Focus moves to options instead of staying on input. No `aria-live` for result count. WCAG 4.1.3 concern.
- **Suggestion:** Add IDs to options, track active index, set `aria-activedescendant`. Add visually-hidden `aria-live="polite"` region.

### AUDIT-2026-0015: Switch without label prop has no accessible name
- **Target:** Switch.tsx:63
- **Description:** When `label` is not provided, bare button with `role="switch"` but no `aria-label`. WCAG 4.1.2 violation.
- **Suggestion:** Add `aria-label` prop. Dev warning when neither `label` nor `aria-label` is provided.

### AUDIT-2026-0016: Popover panel missing focus management
- **Target:** Popover.tsx:57
- **Description:** Has `role="dialog"` but no `useFocusTrap` or `useFocusRestore`. WCAG 2.4.3 violation.
- **Suggestion:** Apply `useFocusTrap(panelRef, open)` and `useFocusRestore(open)` as in Dialog.tsx.

### AUDIT-2026-0017: Dropdown trigger wrapper not keyboard-focusable
- **Target:** Dropdown.tsx:59
- **Description:** Trigger div has onClick/onKeyDown but no `tabIndex`. Not reachable via keyboard.
- **Suggestion:** Add `tabIndex={0}` and `role="button"`, or render as `<button>`.

### AUDIT-2026-0018: Popover trigger wrapper not keyboard-focusable
- **Target:** Popover.tsx:108
- **Description:** Trigger div has onClick but no tabIndex, role, or onKeyDown. No Enter/Space support.
- **Suggestion:** Add `tabIndex={0}`, `role="button"`, and onKeyDown handler.

### AUDIT-2026-0019: Tooltip not accessible to keyboard-only users
- **Target:** Tooltip.tsx:47
- **Description:** Wrapper relies on onFocus/onBlur but has no tabIndex. No `aria-describedby` linking trigger to content.
- **Suggestion:** Add `tabIndex={0}` for non-focusable children. Generate ID, apply `aria-describedby` via cloneElement.

### AUDIT-2026-0020: Toast/Snackbar close buttons below 44px touch target
- **Target:** Toast.tsx:274, Snackbar.tsx:121
- **Description:** Close button is 28x28px (w-7 h-7), below WCAG 2.5.8 minimum of 44x44px.
- **Suggestion:** Increase to w-11 h-11, or use padding/transparent hit area.

### AUDIT-2026-0021: Chip uses span instead of button, not keyboard accessible
- **Target:** Chip.tsx:35
- **Description:** `<span>` with cursor-pointer, not focusable, no Enter/Space response. Delete button is only 14x14px.
- **Suggestion:** Change to `<button>`. Increase delete button to w-6 h-6 (24x24px).

### AUDIT-2026-0022: Avatar missing meaningful alt text and status dot label
- **Target:** Avatar.tsx:44
- **Description:** No alt prop defaults to `alt=""` (decorative). Status dot has no accessible label.
- **Suggestion:** Default `alt={initials || 'User avatar'}`. Add sr-only text to status dot.

### AUDIT-2026-0023: Pagination missing aria-current="page"
- **Target:** Pagination.tsx:33
- **Description:** Active prop changes visual style but does not set `aria-current="page"`.
- **Suggestion:** Add `aria-current={active ? 'page' : undefined}`.

### AUDIT-2026-0024: Table th elements missing scope attribute
- **Target:** Table.tsx:43
- **Description:** `<th>` rendered without `scope="col"`. Screen readers need explicit scope.
- **Suggestion:** Add `scope="col"` as default. Consider a scope prop.

### AUDIT-2026-0025: Stepper lacks ARIA labeling for step state
- **Target:** Stepper.tsx:120
- **Description:** Step state (completed/active/pending) conveyed only visually. No group role on Stepper.
- **Suggestion:** Add `aria-label` per step with state. Wrap in `role="group"`. Add sr-only text for checkmark.

### AUDIT-2026-0026: Chart components have generic non-descriptive aria-label
- **Target:** Chart.tsx:132
- **Description:** "Bar chart", "Line chart" labels provide zero data information.
- **Suggestion:** Accept custom `aria-label`/title prop. Add `<desc>` in SVG. Consider visually-hidden table.

### AUDIT-2026-0027: AnimatedToggle missing aria-pressed attribute
- **Target:** AnimatedToggle.tsx:18
- **Description:** Toggle button has no `aria-pressed`, `aria-checked`, or `aria-label`.
- **Suggestion:** Add `aria-pressed={checked}`. Consider requiring `aria-label` prop.

### AUDIT-2026-0028: DropdownItem focus:outline-none without visible replacement
- **Target:** Dropdown.tsx:121
- **Description:** `focus:outline-none` replaced with background-color-only change. May not meet WCAG 2.4.7.
- **Suggestion:** Add `focus-visible:ring-2 focus-visible:ring-[var(--ring)]`.

## Medium Issues

### AUDIT-2026-0029: Decorative SVGs missing aria-hidden
- **Target:** Select.tsx:66, Checkbox.tsx:51/63, Breadcrumb.tsx:48
- **Description:** Decorative SVGs lack `aria-hidden="true"`. Combobox has the correct pattern.
- **Suggestion:** Add `aria-hidden="true"` to all decorative SVG elements.

### AUDIT-2026-0030: Hardcoded text-white in destructive variants
- **Target:** Button.tsx:15, Badge.tsx:12
- **Description:** `text-white` instead of design token. Won't adapt to themes.
- **Suggestion:** Replace with `text-[var(--error-foreground)]`.

### AUDIT-2026-0031: Hardcoded rounded-[6px] instead of design token
- **Target:** Button.tsx:18, Toast.tsx:265, Snackbar.tsx:113
- **Description:** Uses `rounded-[6px]` while other variants use `var(--radius-*)` tokens.
- **Suggestion:** Replace with `rounded-[var(--radius-sm)]`.

### AUDIT-2026-0032: Hardcoded rgba shadow values
- **Target:** Toast.tsx:67, Snackbar.tsx:11, Slider.tsx:29
- **Description:** Hardcoded rgba shadows don't adapt to theme changes.
- **Suggestion:** Define shadow tokens or use CSS custom properties.

### AUDIT-2026-0033: Dialog hardcoded w-[360px] with no responsive breakpoint
- **Target:** Dialog.tsx:55
- **Description:** Fixed width overflows on viewports < 360px.
- **Suggestion:** Add `max-w-[90vw]` alongside fixed width.

### AUDIT-2026-0034: Tooltip always rendered in DOM when invisible
- **Target:** Tooltip.tsx:49
- **Description:** Always present with `role="tooltip"`, toggled via opacity. Screen readers read invisible content.
- **Suggestion:** Conditionally render or add `aria-hidden={!visible}`.

### AUDIT-2026-0035: Toast/Snackbar action button only dismisses
- **Target:** Toast.tsx:261, Snackbar.tsx:108
- **Description:** Action button onClick only dismisses. No way to pass user-defined callback (e.g., "Undo").
- **Suggestion:** Add `onAction?: () => void` field.

### AUDIT-2026-0036: Input rightIcon wrapper has cursor-pointer but may not be interactive
- **Target:** Input.tsx:50
- **Description:** Hardcoded `cursor-pointer` on potentially decorative icon. No button role or keyboard handling.
- **Suggestion:** Remove hardcoded cursor-pointer. Provide `onRightIconClick` if interactivity needed.

### AUDIT-2026-0037: Tabs and Accordion missing keyboard arrow navigation
- **Target:** Tabs.tsx:81, Accordion.tsx:75
- **Description:** WAI-ARIA patterns require arrow key navigation between triggers. Currently only click events.
- **Suggestion:** Add onKeyDown for ArrowLeft/Right (Tabs), ArrowUp/Down (Accordion). Implement roving tabindex.

### AUDIT-2026-0038: TabContent removed from DOM when inactive
- **Target:** Tabs.tsx:127
- **Description:** Returns null when not active, destroying form inputs, scroll positions, and component state.
- **Suggestion:** Use `hidden={true}` and `aria-hidden={true}` instead of returning null.

### AUDIT-2026-0039: Accordion content has open animation but no close animation
- **Target:** Accordion.tsx:106
- **Description:** Returns null when closed — expand-in animation exists but content disappears instantly on close.
- **Suggestion:** Keep in DOM, animate with `data-state="open"/"closed"` CSS.

### AUDIT-2026-0040: Divider not using semantic hr element
- **Target:** Divider.tsx:42
- **Description:** Renders as `<div>`, screen readers won't announce as separator.
- **Suggestion:** Use `<hr role="separator" />` for non-labeled variants.

### AUDIT-2026-0041: CodeBlock copy button lacks SR feedback and error handling
- **Target:** CodeBlock.tsx:28
- **Description:** No `aria-live` for "Copied!" state. No `.catch()` on clipboard promise.
- **Suggestion:** Add `aria-live="polite"`. Handle clipboard errors.

### AUDIT-2026-0042: EmptyState has no ARIA role
- **Target:** EmptyState.tsx:19
- **Description:** Plain div with no semantic cue for "no data" status.
- **Suggestion:** Add `role="status"`.

### AUDIT-2026-0043: Combobox option items have small touch targets
- **Target:** Combobox.tsx:215
- **Description:** Options ~36px tall, below 44px WCAG 2.5.8 recommendation.
- **Suggestion:** Increase to `py-3` or `min-h-[44px]`.

### AUDIT-2026-0044: Checkbox disabled state visually inconsistent
- **Target:** Checkbox.tsx:33
- **Description:** Disabled checkboxes look nearly identical to enabled. Other components use `disabled:opacity-50`.
- **Suggestion:** Add `peer-disabled:opacity-60` to outer label wrapper.

### AUDIT-2026-0045: DropdownDivider missing role="separator"
- **Target:** Dropdown.tsx:98
- **Description:** Plain div inside `role="menu"`, not announced as group separator.
- **Suggestion:** Add `role="separator"` or use `<hr>`.

### AUDIT-2026-0046: Indicator sets aria-valuemin/max unnecessarily
- **Target:** Indicator.tsx:27
- **Description:** Indeterminate spinner should omit all three ARIA value attributes per spec.
- **Suggestion:** Remove `aria-valuemin` and `aria-valuemax`. Add `aria-label="Loading"`.
